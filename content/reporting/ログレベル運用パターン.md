## 概要

各ログレベルの使い分けを具体的なコード例で示し、チーム全体で統一したログ運用を実現するパターン

## 良い例

## ログレベル運用ガイド

### ERROR（即時対応が必要）
```javascript
logger.error('Payment processing failed', {
  orderId: order.id,
  errorCode: err.code,
  message: err.message
});
```
→ アラート通知 → オンコール対応

### WARN（要注意だが処理継続）
```javascript
logger.warn('API response slow', {
  endpoint: '/api/users',
  responseTime: 2500,
  threshold: 2000
});
```
→ ダッシュボード監視 → 傾向分析

### INFO（正常な業務イベント）
```javascript
logger.info('User login successful', {
  userId: user.id,
  method: 'oauth',
  timestamp: new Date().toISOString()
});
```
→ 監査ログ → 利用統計

### DEBUG（開発時のみ）
```javascript
logger.debug('Cache lookup', {
  key: cacheKey,
  hit: !!cachedValue
});
```
→ 本番ではOFF → 必要時のみ一時的にON

## 悪い例

全てのログを `logger.info()` で出力。重要度が区別できず、障害時に ERROR ログだけでフィルタリングできない

## 使いどころ

チームのログ規約策定、コードレビュー、障害対応の効率化
