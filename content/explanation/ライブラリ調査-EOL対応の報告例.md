## 説明

出典: 技術負債対応プロジェクト / レベル: 中級

## 悪い例 1

今使っているライブラリが古いので新しいのに変えたいです。いくつか候補があります。どれがいいですかね。

## ライブラリ移行調査報告

### 1. 調査背景
- 現行: moment.js v2.29（2024/9 EOL済み、セキュリティパッチ停止）
- 影響: 日付処理全般（API/フロント共通、使用箇所142ファイル）
- リスク: CVE未対応、Node.js新バージョンとの互換性なし

### 2. 候補と比較
| 評価軸 | date-fns | dayjs | Temporal(Stage3) |
|---|---|---|---|
| バンドルサイズ | 12KB(tree-shake) | 2KB | 0KB(native) |
| API互換性 | 低（関数型） | 高（moment互換） | 低 |
| 移行コスト | 高（142箇所書換） | 低（ほぼ置換） | 高（Polyfill必要） |
| 長期安定性 | ◎（活発） | ○（安定） | ◎（標準化予定） |
| TypeScript | ◎（型付き） | ○ | ◎ |

### 3. 推奨
**dayjs を推奨**
- 理由: moment.js とAPI互換があり移行コスト最小
- 移行方針: import 置換 + API差分箇所のみ修正
- 見積もり: 3人日（置換2日 + テスト1日）
- フォールバック: 問題発生時は date-fns に切り替え

## 解説 1

- **改善点**: 「古いので新しいのに変えたい」 -- 何が古いのか、なぜ変える必要があるのか（EOL/セキュリティ）が不明
- **改善点**: 「いくつか候補があります」 -- 候補名・比較軸・推奨理由がない。「どれがいいですかね」では判断できない
- **スキル**: 「2024/9 EOL済み」 -- EOL日付を明示し、セキュリティリスクを具体化。「古い」を定量化している
- **スキル**: 「使用箇所142ファイル」 -- 影響範囲を数値で示す。移行コストの根拠になる
- **スキル**: 「フォールバック: 問題発生時はdate-fnsに切り替え」 -- 推奨案がダメだった場合の代替案を用意。リスク管理として重要

## 悪い例 2

Node.jsのバージョンを上げたいです。今のバージョンが古いので。テストは全部通ったので大丈夫だと思います。

## Node.js バージョンアップ計画

### 1. 現状と緊急度
- 現行: Node.js 18.x（EOL: 2025/4/30 → **既にEOL超過**）
- 目標: Node.js 22.x LTS（EOL: 2027/4、Active LTS中）
- 緊急度: **高**（セキュリティパッチが停止済み）

### 2. 影響調査結果
- 依存パッケージ: 全187パッケージ中、非互換 **3件**
  1. `node-sass` → `sass`（Dart Sass）に置換が必要
  2. `sharp` v0.30 → v0.33にアップデート
  3. `bcrypt` v5.0 → v5.1にアップデート（ネイティブリビルド）
- CI/CD: GitHub Actions の `setup-node` バージョン更新が必要
- Docker: ベースイメージ `node:18-alpine` → `node:22-alpine`

### 3. 移行手順
1. ブランチ作成 + Node 22 に切り替え（0.5h）
2. 非互換パッケージ3件のアップデート（2h）
3. 全テスト実行 + 差分確認（1h）
4. ステージング環境でのE2Eテスト（1h）
5. 本番デプロイ + モニタリング（0.5h）

### 4. ロールバック計画
- Docker イメージの前バージョンに即時切り戻し可能
- 切り戻し手順: `kubectl rollout undo deployment/api`

## 解説 2

- **改善点**: 「バージョンを上げたいです」 -- 現行バージョン、目標バージョン、なぜ上げる必要があるかが不明
- **改善点**: 「大丈夫だと思います」 -- 「思います」ではなく、非互換パッケージの調査結果を示す
- **スキル**: 「既にEOL超過」 -- EOLの日付と現在の状況を明示し、緊急度を客観的に示す
- **スキル**: 「全187パッケージ中、非互換 3件」 -- 影響範囲を調査して定量化。「大丈夫」を根拠付きで示している
- **スキル**: 「### 4. ロールバック計画」 -- バージョンアップには必ずロールバック計画を添える。「ダメだったら戻せる」安心感
