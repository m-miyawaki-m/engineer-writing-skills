## 説明

ログに機密情報を出力しないこと。パスワード・トークン・個人情報・クレジットカード番号等がログに残ると、ログ流出時に重大なセキュリティインシデントになる

## 具体例

【出してはいけないもの】
- パスワード（平文・ハッシュ含む）
- アクセストークン / リフレッシュトークン
- 個人情報（住所、電話番号、メール※場合による）
- クレジットカード番号
- APIキー / シークレット

【マスキングの例】
```javascript
// NG
logger.info('Login attempt', { email, password });
// OK
logger.info('Login attempt', { email, password: '***' });
// OK（そもそもpasswordをログに渡さない）
logger.info('Login attempt', { email });
```

## 悪い例

デバッグ中に `console.log(req.body)` でリクエスト全体を出力 → パスワードやトークンがログに残る

## 活用場面

コードレビュー、セキュリティ監査、ログ設計ガイドライン策定
