## 概要

出典: 社内技術導入プロジェクト / レベル: 上級

## 悪い例 1

上司に「うちもAI導入したい」と言われたので、ChatGPTについて2ヶ月調べました。
いろいろ記事を読みましたが、結局よくわかりませんでした。
導入は難しそうなので見送りにしようと思います。

## 良い例 1

## AI導入 調査設計書（Week0）

【調査目的】カスタマーサポートへのAI導入の実現可能性を評価する
【背景】月間問い合わせ500件、1件平均15分、人件費換算で月50万円
【調査範囲】
  1. 現状の課題整理（ヒアリング）
  2. AI技術の選択肢と比較
  3. コスト・工数の概算
  4. リスクと対策
【期限】4週間（中間報告: Week2金曜）
【成果物】調査報告書 + Go/No-Go判断材料 + Go時のWBS案

## 解説 1

- **改善点**: 「2ヶ月調べました」 -- 調査設計なし、期限なし、中間報告なし。時間だけ過ぎる典型パターン
- **改善点**: 「いろいろ記事を読みました」 -- 調査の範囲・方法が定義されていない。何を読んで何がわかったのか不明
- **改善点**: 「結局よくわかりませんでした」 -- ゴールが曖昧だったので「わかった/わからない」の判断基準がない
- **スキル**: 「【調査目的】」 -- 調査も「プロジェクト」として設計する。目的・範囲・期限・成果物を明示
- **スキル**: 「月間問い合わせ500件」 -- 現状を定量化することで、改善効果の測定基準ができる
- **スキル**: 「中間報告: Week2金曜」 -- 中間チェックポイントで軌道修正できる。2ヶ月ブラックボックスにしない

## 悪い例 2

ChatGPT、Claude、Geminiを試しました。ChatGPTが一番よさそうです。導入しましょう。

## 良い例 2

## 技術選択肢の比較（Week2 中間報告）

【候補】
  A. ChatGPT API + RAG構成（自社構築）
  B. チャットボットSaaS（Intercom / Zendesk AI）
  C. 社内FAQ検索エンジン強化（Elasticsearch + ベクトル検索）

【評価軸と比較】
  | 軸 | A. GPT+RAG | B. SaaS | C. ES強化 |
  |---|---|---|---|
  | 初期コスト | 中（100万） | 高（月30万〜） | 低（20万） |
  | 回答精度 | 高（推定） | 中 | 中 |
  | 運用負荷 | 中 | 低 | 低 |
  | カスタマイズ性 | 高 | 低 | 中 |
  | 導入期間 | 2ヶ月 | 2週間 | 1ヶ月 |

【推奨】A案を推奨。理由: 精度とカスタマイズ性が最重要。ただしPoCで精度を検証してから判断
【リスク】精度がPoC基準（70%）を下回った場合はB案にフォールバック

## 解説 2

- **改善点**: 「一番よさそうです」 -- 比較軸なし、根拠なし。「よさそう」は主観であり判断材料にならない
- **スキル**: 「【評価軸と比較】」 -- 複数案を同一軸で比較することで客観的な判断が可能になる
- **スキル**: 「PoCで精度を検証してから判断」 -- 「推定」で終わらせず、PoCで事実を確認するステップを入れる
- **スキル**: 「B案にフォールバック」 -- リスク対策として代替案を用意。失敗しても次の手がある

## 悪い例 3

AIを試してみました。なかなかいい感じでした。本番導入しましょう。

## 良い例 3

## PoC結果報告 + 次アクション（Week3末）

【PoC概要】GPT-4 API + RAG構成でFAQ上位50件を対象に検証
【結果】
  - 正答率: 全体78%（定型FAQ 92%、複合質問 40%）
  - 応答速度: 平均1.2秒（基準3秒以内 ✅）
  - コスト: 0.8円/件（人件費比95%削減）

【考察】
  - 定型FAQ（全体の70%）は実用レベル → すぐ自動化可能
  - 複合質問（全体の30%）は精度不足 → 人間にエスカレーション
  - 段階的導入が現実的

【Go/No-Go判断】Go（段階的導入）を推奨

【本実装スケジュール案】
  M1 (Week6末): MVP完成（定型FAQのみ自動応答）
  M2 (Week8末): 社内テスト + フィードバック収集
  M3 (Week10末): 本番リリース（定型FAQのみ）
  M4 (Week14末): 複合質問対応の改善PoC

【ナレッジ蓄積】
  - PoC設計テンプレート → Wiki登録済み
  - RAG構成のチャンクサイズ知見 → 技術Wikiに記載
  - 「PoCは3日以内」ルール → チーム調査ガイドラインに追加

## 解説 3

- **改善点**: 「なかなかいい感じ」 -- 数値なし、基準なし。「いい感じ」では意思決定できない
- **スキル**: 「正答率: 全体78%」 -- 全体数値だけでなく、カテゴリ別に分解することで具体的な改善策が見える
- **スキル**: 「段階的導入が現実的」 -- 0か100かではなく、段階的アプローチを提案。リスクを抑えつつ価値を出す
- **スキル**: 「【ナレッジ蓄積】」 -- プロジェクト完了時に知見を文書化し、次のプロジェクトで再利用可能にする
