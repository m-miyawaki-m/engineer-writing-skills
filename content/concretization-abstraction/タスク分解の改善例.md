## 説明

出典: スプリント計画チケット / レベル: 中級

## 悪い例

検索機能を作る。2〜3日くらいかかりそう。

## 具体例

【目的】商品検索機能の実装
【前提】Elasticsearch 8.x 導入済み、API は FastAPI

【タスク分解】
1. インデックス定義・マッピング作成（0.5h / ブロッカーなし）
2. 検索APIエンドポイント実装（2h / 1に依存）
3. フィルタ・ソート・ページネーション（1.5h / 2に依存）
4. フロント検索UI（2h / ブロッカーなし ※2と並行可能）
5. 結合テスト（1h / 3,4に依存）

【合計見積もり】7h（バッファ込み 1日）
【完了条件】全テスト通過 + レビュー承認 + ステージング動作確認

## 解説

- **改善点**: 「2〜3日くらい」 -- 根拠のない概算。分解して積み上げた見積もりにすべき
- **改善点**: 「検索機能を作る」 -- タスクが大きすぎて進捗が見えない。1PR単位に分割すべき
- **スキル**: 「ブロッカーなし ※2と並行可能」 -- 依存関係を明示することで、並行作業の可能性が見える
- **スキル**: 「【完了条件】」 -- 何をもって「完了」とするかを事前に定義。認識齟齬を防ぐ
