## 説明

新人〜ジュニアレベルで身につけるべきスキルを段階的に整理し、各段階の到達目標を明確にするパターン

## 具体例

### Phase1: 入社〜3ヶ月（生存期）
- [ ] 報連相を適切なタイミングで行える
- [ ] 30分詰まったら質問できる（質問の型で）
- [ ] 既存コードを読んで修正箇所を特定できる
- [ ] PRを書いてレビューを受けられる
- [ ] 日報を毎日書ける

### Phase2: 3〜6ヶ月（貢献期）
- [ ] タスクの見積もりを根拠付きで出せる
- [ ] 小〜中規模の機能を1人で実装できる
- [ ] テスト設計（UT/FT）を自分でできる
- [ ] 設計レビューで質問・指摘ができる
- [ ] 障害報告を構造化して書ける

### Phase3: 6ヶ月〜1年（自走期）
- [ ] スコープ管理と関係者調整ができる
- [ ] 技術選定の比較・提案ができる
- [ ] 後輩への簡単な説明・レビューができる
- [ ] 週報で傾向分析と提案ができる
- [ ] 設計書を書いてレビューを通せる

## 悪い例

「3年で一人前になってね」（具体的なマイルストーンなし）

## 活用場面

新人育成計画、1on1での目標設定、自己評価、メンタリング
